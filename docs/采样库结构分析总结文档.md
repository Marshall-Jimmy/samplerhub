| 标注模式 | 示例 | 说明 |

| `数字bpm` | `140bpm`, `94bpm` | 最常见，小写 |

## 五、文件夹命名模式总结

### 模式 A：专业商业采样包（Black Octopus）

```
[乐器类型] - [子分类]
```
- 使用连字符 `-` 分隔
- 子分类固定词汇：`Loops`, `One Shots`, `Fills`, `Top Loops`, `Percussion Loops`
- 乐器类型固定词汇：`Bass`, `Drum`, `Synth`, `FX`, `Vocal`, `Live Instrument`

### 模式 B：个人制作人收藏（@wolley stash）

```
[乐器类型]/（纯英文单词）
```
- 乐器类型：`808s`, `Claps`, `FX`, `Hi-Hats`, `Kicks`, `Open Hats`, `Snares`
- 复数形式为主

### 模式 C：序号编号系统（Osaka Sound）

```
[前缀]_[类型]_[序号].wav
```
- 无子分类文件夹

### 模式 D：多风格综合合集（Guozipeng）

```
[风格名] Multi Kit/ 或 [艺人名] Sound Kit/
```
- 内部按乐器类型分文件夹
- 文件名模式多样且不统一

### 模式 E：按歌曲组织（TylerTheCreator）

```
[歌曲编号] [歌曲名]/
```
- 按专辑曲目结构组织

### 6.2 二级分类（Loop vs One Shot）

| 分类 | 判断依据 |
|------|---------|
| **Loop** | 文件名含 `Loop`、`loop`；文件夹名含 `Loops`；文件时长 > 1秒 且 文件名含 BPM |
| **One Shot** | 文件名含 `One Shot`、`Oneshot`、`One_Shot`；文件夹名含 `One Shots`；文件时长 < 2秒 |
| **Fill** | 文件名含 `Fill`、`Drumfill` |
| **Top Loop** | 文件名含 `Top`、`No Kick`、`No_Kick` |

### 6.3 元数据提取规则

| 元数据 | 提取正则 | 示例 |
|--------|---------|------|
| **BPM** | `(\d{2,3})\s*[Bb][Pp][Mm]` | `140bpm` → 140 |
| **调性** | `[_\s]([A-G][#b]?)(min|maj|m|major|minor|M)?[_\s.\)]` | `_A_` → A, `_Fm_` → Fm |
| **编号** | `[_\s](\d{1,3})[_\s.\)]` 或 `\((\d{1,3})\)` | `_01` → 01, `(3)` → 3 |
| **制作人** | `@(\w+)` | `@kolaloibeat` → kolaloibeat |
| **系列名** | 文件名第一个下划线前的部分 | `Leviathan4_Bass...` → Leviathan4 |

### 6.4 特殊处理规则

1. **圆括号内数字 ≠ BPM**：`(113)` 是变体编号，不是 BPM
2. **808 归类**：文件名含 `808` 应归入 808/Sub Bass 类，而非普通 Bass
3. **MIDI 文件**：根据所在文件夹名判断对应乐器类型
4. **预设文件**：根据扩展名（`.fxp`, `.preset`）和所在文件夹名判断对应插件
5. **制作人标签**：`@xxx` 格式可用于标记来源/风格标签
6. **方括号类型**：`[808]`, `[Clap]`, `[Kick]` 等方括号内容可作为分类依据
7. **嵌套采样包**：Guozipeng 合集中的子套件名（如 `BNYX Multi Kit`）可作为风格标签

## 七、已知问题和边缘情况

1. **命名不统一**：不同采样包使用完全不同的命名规范，无法用单一规则覆盖
2. **BPM 标注率低**：仅 2.5% 的文件明确标注 BPM，大部分需要通过音频分析获取
3. **格式混杂**：Nujabes Kit 全部为 MP3，其余为 WAV/AIF，需统一处理
4. **深层嵌套**：Guozipeng 合集嵌套达 5+ 层，需递归扫描
5. **非音频文件**：包含大量预设文件（Serum、Effectrix、Looperator 等），需区分处理
6. **中文文件夹名**：`民族采样音色` 为中文命名，需支持 Unicode
7. **特殊字符**：文件名中包含 `✦`、`™`、`№`、`#`、`!` 等特殊字符
8. **空格 vs 下划线**：同一采样包内混用空格和下划线分隔符
9. **大小写不一致**：同一类型可能有 `Hi-Hat`、`hihat`、`Hi Hat`、`HH` 等多种写法
